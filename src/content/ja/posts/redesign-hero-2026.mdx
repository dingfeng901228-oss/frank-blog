前のヒーローを見返すと、寂しいの一言だった。

タイトルとタグラインがぽつんと置いてあるだけで、「自分が何を書いている人なのか」が一行目では伝わらない。読みに来てくれた人の第一声が「ここ、誰のサイトだっけ」になっていたら、かなりもったいない。

だから今回は、**深色科技风（ダーク・テック系）の雰囲気はそのままに、可読性と視覚的階層を一段引き上げる**ことを目標に、ヒーローセクションを再設計した。

## 1. 背景：純黒をやめ、深蓝＋紫蓝のグラデーションへ

元は `#0A0A0F`（ほぼ純黒）だった。悪くはないが、テキストが前に出すぎて、画面全体が「重い」。

今風のやり方は、**ベースは暗いが完全に黒ではない**色味を作る。具体的には：

- ベース：`#0F1428`（深蓝）
- 上半分：`#1A1B3A`（紫寄り）にフェード
- 中央上部に**ラジアルグラデーションで青い柔らかな光**を一枚重ねる

これだけで、画面の奥ゆきが全然変わる。光の方向と強さが暗示されるので、視線がタイトルに自然に向かう。

```css
background-image:
  radial-gradient(ellipse 80% 60% at 50% 0%,
    rgba(139, 92, 246, 0.22) 0%,
    rgba(59, 130, 246, 0.10) 35%,
    rgba(15, 20, 40, 0) 70%),
  radial-gradient(ellipse 60% 50% at 50% 30%,
    rgba(59, 130, 246, 0.14) 0%,
    rgba(15, 20, 40, 0) 60%),
  linear-gradient(180deg, #0F1428 0%, #141937 40%, #0F1428 100%);
```

ポイントは**「完全に黒」を絶対に使わない**こと。純黒は LED 画面だと目が疲れるし、その上に何を乗せても単調に見える。

## 2. タイトル：純白＋軽い発光で「第一視線」を作る

ダーク系で一番よくやる失敗は、タイトルも `#E8E8EC` みたいな「白っぽい色」で止めてしまうこと。背景が暗いと少し明るくしただけで十分な気がしてしまうが、**それでは周囲と区別がつかない**。

今回は：

- カラー：`#FFFFFF`（純白）
- ウェイト：`font-weight: 600-700`（semibold）
- トラッキング：`tracking-tight`（詰める）
- 発光：`text-shadow: 0 0 40px rgba(59,130,246,0.25)`

```tsx
<h1
  className="font-serif text-5xl md:text-7xl font-semibold tracking-tight mb-6 text-white"
  style={{
    textShadow:
      '0 0 40px rgba(59, 130, 246, 0.25), 0 0 80px rgba(139, 92, 246, 0.12)',
  }}
>
  {hero.title}
</h1>
```

発光は**青＋紫の二段重ね**にした。片方だけだと「ネオンっぽい」になりすぎるが、二色で柔らかく混ぜると、上品に光る感じになる。Vercel のキービジュアルがまさにこの手法。

## 3. サブタイトル：白 75% で「読ませる」

サブタイトル（タグライン）はタイトルより**少し落とす**。コントラスト比を意識して：

```css
color: rgba(255, 255, 255, 0.75);
```

純白だと主張が強すぎて、タイトルとの主従が崩れる。75% くらいが「読もうと思えば読める、でも目を奪わない」ちょうど良いライン。

## 4. CTA ボタン：主＋次の二段構え

「何がしたいサイトか」を最初の一行で伝えるために、**アクションを 2 つ**並べた：

- **主ボタン：「ブログを読む」** → グラデーション青（`#3B82F6` → `#6366F1`）、ホバーで発光が強くなる＋ 1px 持ち上がる
- **次ボタン：「作ったもの」** → ガラス風（半透明＋ボーダー＋ backdrop-blur）、ホバーで青の縁取り

```css
.hero-cta-primary {
  background: linear-gradient(135deg, #3B82F6 0%, #6366F1 100%);
  box-shadow:
    0 4px 20px rgba(59, 130, 246, 0.35),
    0 0 0 1px rgba(255, 255, 255, 0.08) inset;
}
.hero-cta-primary:hover {
  box-shadow:
    0 8px 32px rgba(59, 130, 246, 0.55),
    0 0 0 1px rgba(255, 255, 255, 0.16) inset;
  transform: translateY(-1px);
  filter: brightness(1.08);
}
```

ホバー時の `translateY(-1px)` は地味だが効く。Raycast や Linear のボタンがみんなこの「持ち上がる」動きを採用しているのは、ちゃんと理由がある。

## 5. ナビゲーション：白 78% ＋ ホバーで青

```css
.nav-link { color: rgba(255, 255, 255, 0.78); }
.nav-link:hover { color: #3B82F6; }
.nav-link[data-active='true'] { color: #FFFFFF; }
```

前は `opacity-70` で透明度を下げていたが、**色そのものを明るくする**ほうがコントロールしやすい。ホバー時の青はアクセントと同じ `#3B82F6` で統一して、配色の一貫性を保つ。

## 全体の話

参考にしたのは Vercel・Linear・Raycast の 3 つ。**「暗いが黒ではない」「光は一点集中」「白は純白で一点だけ」**——この 3 つを守れば、ダークでも「安っぽい」になりにくい。

ダーク系で一番怖いのは、**何も考えずに色を重ねると全部が同じ重さになって、画面がのっぺりする**こと。光・影・色の 3 層を意識して、最初の一目でどこに視線を誘導するか決めてから組み立てると、結果が全然違う。

このページに来てくれた人が、**最初の 1 秒で「あ、しゃれな人だ」と思ってくれたら勝ち**。今回の再設計は、その 1 秒のためにある。
