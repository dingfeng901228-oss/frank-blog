回头看之前的 Hero，只有一句话：太空了。

标题和副标题孤零零地立着，"这是一个什么样的人写的站"——这一眼完全没传递出去。如果访客第一反应是"这是谁的站来着"，那基本就流失了。

所以这次的目标是：**保留深色科技风的氛围，但把可读性和视觉层次往上提一档**。

## 1. 背景：放弃纯黑，用深蓝＋紫蓝渐变

之前是 `#0A0A0F`（几乎纯黑）。不丑，但文字会显得"凸出来"，整个画面很"重"。

现代的做法是：**底色暗，但绝对不是黑**。具体来说：

- 底色：`#0F1428`（深蓝）
- 上半部：向 `#1A1B3A`（偏紫）渐变
- 中央顶部叠一层**径向渐变的柔光**

这三层一叠，屏幕的纵深完全不同了。光的方向和强度有了暗示，视线会自然被引到标题。

```css
background-image:
  radial-gradient(ellipse 80% 60% at 50% 0%,
    rgba(139, 92, 246, 0.22) 0%,
    rgba(59, 130, 246, 0.10) 35%,
    rgba(15, 20, 40, 0) 70%),
  radial-gradient(ellipse 60% 50% at 50% 30%,
    rgba(59, 130, 246, 0.14) 0%,
    rgba(15, 20, 40, 0) 60%),
  linear-gradient(180deg, #0F1428 0%, #141937 40%, #0F1428 100%);
```

关键：**永远不要用纯黑**。纯黑在 LED 屏上很伤眼，而且任何东西叠上去都显得单调。

## 2. 标题：纯白＋轻微发光，"第一眼焦点"

深色系最常犯的错，是把标题也做成 `#E8E8EC` 这种"白但不是纯白"。背景一暗，亮度稍微调高就觉得够了——**但这样跟周围没区分**。

这次：

- 颜色：`#FFFFFF`（纯白）
- 字重：`font-weight: 600-700`（semibold）
- 间距：`tracking-tight`（收紧）
- 发光：`text-shadow: 0 0 40px rgba(59,130,246,0.25)`

```tsx
<h1
  className="font-serif text-5xl md:text-7xl font-semibold tracking-tight mb-6 text-white"
  style={{
    textShadow:
      '0 0 40px rgba(59, 130, 246, 0.25), 0 0 80px rgba(139, 92, 246, 0.12)',
  }}
>
  {hero.title}
</h1>
```

发光用了**蓝＋紫双层**。单色会显得"霓虹灯感太强"，两色柔和混在一起，就是那种"高级地发着光"的感觉。Vercel 的主视觉就是这么做的。

## 3. 副标题：白色 75%，"能读但不抢戏"

副标题比标题**要压一档**。控制对比度：

```css
color: rgba(255, 255, 255, 0.75);
```

纯白的话会太抢，主从关系就乱了。75% 正好是"想读就读得到，但不抢眼"的那条线。

## 4. CTA 按钮：主＋次双层结构

为了让访客一眼明白"这个站能做什么"，放了**两个动作**：

- **主按钮：「阅读博客」** → 渐变蓝（`#3B82F6` → `#6366F1`），hover 时发光加强＋上浮 1px
- **次按钮：「作品集」** → 玻璃风（半透明＋描边＋ backdrop-blur），hover 时蓝色描边

```css
.hero-cta-primary {
  background: linear-gradient(135deg, #3B82F6 0%, #6366F1 100%);
  box-shadow:
    0 4px 20px rgba(59, 130, 246, 0.35),
    0 0 0 1px rgba(255, 255, 255, 0.08) inset;
}
.hero-cta-primary:hover {
  box-shadow:
    0 8px 32px rgba(59, 130, 246, 0.55),
    0 0 0 1px rgba(255, 255, 255, 0.16) inset;
  transform: translateY(-1px);
  filter: brightness(1.08);
}
```

hover 时的 `translateY(-1px)` 看不出来差别，但真的有效。Raycast 和 Linear 的按钮都在用这个"上浮"动效，是有原因的。

## 5. 导航：白 78% ＋ hover 变蓝

```css
.nav-link { color: rgba(255, 255, 255, 0.78); }
.nav-link:hover { color: #3B82F6; }
.nav-link[data-active='true'] { color: #FFFFFF; }
```

之前用 `opacity-70` 压透明度，但**直接控制颜色本身**更可控。hover 时的蓝和主按钮的蓝是同一个 `#3B82F6`，配色一致性有了。

## 整体说几句

参考的是 Vercel、Linear、Raycast 三家。**"暗但不黑""光只集中一点""白只在最关键处用纯白"**——这三条做到，深色系就不容易"显得廉价"。

深色系最怕的是**没想清楚就堆颜色，结果所有元素一样重，画面变得平**。光、影、色三层要有意识，先决定"第一眼往哪里看"，再去组装其余的。

如果来访者**第一秒就觉得"哦，挺有审美的人"**，就赢了。这次改 Hero，就是为这第一秒服务的。
